Feature #11: Answer Auto-Save (Mobile) ✅ HIGH PRIORITY
-------------------------------------------------------
**The Problem:**
- Player fills out 6 answers on phone
- Phone screen turns off (sleep mode)
- Browser app switches to another app
- Player returns → All answers GONE
- Has to re-enter everything from memory

**The Solution:**
- **Automatic localStorage backup** as you type
- Saves every 500ms after you stop typing
- Restores answers when you return
- Tied to specific team code + round ID
- Clears on successful submission

**How It Works:**
```
Player Types:
  "Red" → (500ms pause) → Saved to localStorage
  
Phone Sleeps:
  Screen off for 5 minutes
  
Player Returns:
  Opens page → "✅ Answers restored from auto-save"
  All 6 answers + tiebreaker intact
  
Player Submits:
  Form submits → localStorage cleared
  Next round starts fresh
```

**Storage Key Format:**
```javascript
Key: feud_answers_ABCD_round_3
     └── Team code ┘  └─ Round ID ┘

Stored Data:
{
  "answer1": "Red",
  "answer2": "Blue", 
  "answer3": "Green",
  ...
  "tiebreaker": "45"
}
```

**Features:**
- ✅ Saves on every keystroke (debounced 500ms)
- ✅ Restores on page load
- ✅ Clears on successful submit
- ✅ Tied to specific round (won't restore wrong round)
- ✅ Works across phone sleep/wake
- ✅ Works if app switches
- ✅ Works on page refresh

**Edge Cases Handled:**
- Different rounds don't conflict (separate keys)
- Multiple teams don't conflict (code in key)
- localStorage quota errors caught gracefully
- Old data cleared on submit
- Won't restore if form doesn't exist

**Browser Support:**
- ✅ Chrome Mobile
- ✅ Safari iOS
- ✅ Firefox Mobile
- ✅ Samsung Internet
- ✅ Any modern mobile browser

**Files Modified:**
- templates/play.html (auto-save JavaScript)

**Testing Needed:**

□ Turn screen back on
□ Verify answers still there ✅
□ Switch to another app
□ Return to browser
□ Verify answers still there ✅

Feature #16: "Are You Still There?" Idle Check ✅
--------------------------------------------------
**The Problem:**
- Player joins game
- Gets distracted, leaves phone open
- 45 minutes later, round has changed
- Player confused, game state stale

**The Solution:**
- **30-minute inactivity timer**
- Popup asks: "Are you still there?"
- Resets on ANY user interaction
- Prevents abandoned/stale game states

**How It Works:**
```
Player opens page → Timer starts (30 min)

Player types answer → Timer resets (30 min)
Player scrolls → Timer resets (30 min)
Player taps screen → Timer resets (30 min)

30 minutes of ZERO activity:
  ↓
[Popup appears]
⏰ Are You Still There?

You've been inactive for 30 minutes.

Click OK to continue playing, or Cancel to refresh.

[OK] → Timer resets, game continues
[Cancel] → Page refreshes, gets latest state
```

**Activity Detection:**
Resets timer on:
- Mouse movement
- Mouse clicks
- Keyboard typing
- Scrolling
- Touch events
- Any click/tap

**Features:**
- ✅ 30-minute timeout (configurable)
- ✅ Popup with clear message
- ✅ OK = Continue playing
- ✅ Cancel = Refresh page
- ✅ Timer resets on ANY activity
- ✅ Prevents popup spam (shows once)

**Edge Cases Handled:**
- Multiple popups prevented (flag system)
- Timer cleared on page unload
- Activity during popup resets timer
- Passive event listeners for performance

**Files Modified:**
- templates/play.html (idle detection JavaScript)

**Testing Needed:**

□ Verify popup appears ✅
□ Click OK → Verify timer resets ✅
□ Don't touch for 30 more min
□ Verify popup appears again ✅
□ Click Cancel → Verify page refreshes ✅

**Configuration:**
