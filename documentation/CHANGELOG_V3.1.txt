### **1. WHITE SCREEN ON TIED SCORES** ✅ FIXED

**Bug ID:** #001  
**Severity:** CRITICAL  
**Reported:** User testing  

**Symptom:**
- Clicking "View Round Summary" when two teams have same score
- Results in completely white screen
- No error message visible to user
- Game cannot continue

**Root Cause:**
```python
# In templates/round_summary.html line 202:
({{ 'Over' if ... else 'Under' }} by {{ abs(tiebreaker_info.difference) }})
                                       ^^^^ 
```
- Jinja2 template engine does NOT have built-in `abs()` filter
- Attempting to use undefined filter crashes template rendering
- Error: `jinja2.exceptions.UndefinedError: 'abs' is undefined`

**The Fix:**
```jinja2
OLD (BROKEN):
{{ abs(tiebreaker_info.difference) }}

NEW (FIXED):
{% set abs_diff = tiebreaker_info.difference if tiebreaker_info.difference >= 0 else -tiebreaker_info.difference %}
{{ abs_diff }}
```

**Additional Safety:**
- Added null checks: `{% if tiebreaker_info.difference is not none %}`
- Added existence check: `{% if tied and tiebreaker_info %}`
- Added default values: `{{ tiebreaker_info.tied_score or 0 }}`

**Files Changed:**
- `templates/round_summary.html` (lines 189-211)

**Testing:**
- Scored 2 teams with identical scores (4 points each)
- Clicked "View Round Summary"
- ✅ Page loads correctly
- ✅ Shows tiebreaker info
- ✅ Shows winner
- ✅ Shows difference calculation

### **2. CODE ORDER JUMPING EVERY 10 SECONDS** ✅ FIXED

**Bug ID:** #002  
**Severity:** ANNOYING  
**Reported:** User testing (screenshot provided)  

**Symptom:**
- Team codes display in one order initially
- After 10 seconds (auto-refresh), codes re-sort alphabetically
- Makes it hard to track specific codes
- Visually jarring

**Root Cause:**
```python
# Initial page load (templates/host.html):
SELECT code, used, team_name FROM team_codes ORDER BY id DESC

# Auto-refresh API (app.py /host/codes-status):
SELECT code, used, team_name FROM team_codes ORDER BY code ASC
                                                      ^^^^^^^^
```
- Different sort orders!
- Template sorts by `id DESC` (newest first)
- API sorts by `code ASC` (alphabetical)
- Every refresh re-sorts the list

**The Fix:**
```python
# app.py line 216:
OLD: ORDER BY code ASC
NEW: ORDER BY id DESC
```

**Files Changed:**
- `app.py` (line 216)

**Testing:**
- Loaded host dashboard
- Noted code order
- Waited 10+ seconds for auto-refresh
- ✅ Codes stay in same order
- ✅ No alphabetical re-sorting
- ✅ Flash highlights still work

### **3. MOBILE 3-LINE QUESTIONS CUT OFF FORM** ✅ FIXED

**Bug ID:** #003  
**Severity:** HIGH (Mobile UX)  
**Reported:** User testing on iPhone 16, iPhone 17 Max, Android  

**Symptom:**
- Questions with 2 lines: Work fine
- Questions with 3 lines: Submit button cut off screen
- User cannot submit answers without scrolling
- Tiebreaker field might be partially hidden

**Root Cause:**
```css
/* play.html - Fixed header: */
.header-line2 {
    font-size: 1.8em;  /* TOO BIG on mobile! */
}

body {
    padding-top: 110px;  /* TOO MUCH wasted space */
}
```
- Question text too large (1.8em)
- Line-height too tall (1.3)
- Fixed header takes up ~110px
- 3-line questions = ~150px total
- Leaves no room for form fields

**The Fix:**
```css
@media (max-width: 600px) {
    .header-line2 {
        font-size: 1.1em !important;    /* Smaller */
        line-height: 1.1;                /* Tighter */
    }
    
    .header-line1 {
        font-size: 0.75em !important;   /* Smaller round info */
    }
    
    body {
        padding-top: 80px !important;   /* Less wasted space */
    }
    
    .fixed-header {
        padding: 6px 10px;              /* Tighter padding */
    }
}
```

**Changes:**
- Question font: 1.8em → **1.1em** (39% smaller!)
- Line-height: 1.3 → **1.1** (tighter)
- Body padding: 110px → **80px** (27% less)
- Header padding: Reduced

**Files Changed:**
- `templates/play.html` (lines 307-325, new mobile CSS)

**Testing:**

- ✅ Question readable (not too small)

## 🔍 **TECHNICAL DETAILS**

### **Fix #1 Technical Notes:**

**Jinja2 Built-in Filters:**
Jinja2 has many built-in filters (e.g., `upper`, `lower`, `length`), but `abs` is NOT one of them.

**Available Jinja2 Math:**
- Addition: `{{ a + b }}`
- Subtraction: `{{ a - b }}`
- Multiplication: `{{ a * b }}`
- Division: `{{ a / b }}`
- Modulo: `{{ a % b }}`
- Conditionals: `{% if a > b %}`

**Our Manual abs():**
```jinja2
{% set abs_diff = value if value >= 0 else -value %}
```
This is a standard ternary operation that Jinja2 supports.

---

### **Fix #2 Technical Notes:**

**ORDER BY Behavior:**
- `ORDER BY id DESC` = Newest codes first (id 100, 99, 98...)
- `ORDER BY code ASC` = Alphabetical (ABC1, ABC2, BAJK...)

**Why It Matters:**
- Host generates codes in batches
- Newest codes most likely to be used soon
- Seeing newest first = better UX

**Flash Highlight Compatibility:**
- Flash highlights track by code string
- Works regardless of sort order
- But jumping order makes it confusing

---

### **Fix #3 Technical Notes:**

**Mobile Viewport Sizes:**
- iPhone SE: 375px width
- iPhone 16: 393px width
- iPhone 17 Max: 430px width
- Android varies: 360-420px typical

**Our Breakpoint:**
```css
@media (max-width: 600px)
```
Catches all phones, some tablets in portrait.

**Font Size Math:**
```
Desktop: 1.8em * 16px base = 28.8px
Mobile:  1.1em * 16px base = 17.6px
Savings: 11.2px per line × 3 lines = 33.6px saved!
```

**Readability:**
- 17.6px is still very readable
- iOS minimum recommended: 16px
- Android minimum recommended: 14px
- We're comfortably above both ✅
