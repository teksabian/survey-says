BUG #1: Reset All doesn't invalidate phone sessions
----------------------------------------------------
**Problem:**
- Teams stayed connected after "Reset All" button clicked
- Phones saw new rounds when they shouldn't
- Sessions persisted, not kicked out

**Fix:**
- Added RESET_COUNTER (in-memory global variable)
- Increments every time "Reset All" is clicked
- Teams store reset_counter in session
- Decorator checks reset_counter on every request
- Mismatch = show Game Over page

**Files Modified:**
- app.py:
  * Added RESET_COUNTER global variable (line ~72)
  * Updated team_session_valid decorator to check reset_counter
  * Updated join_submit to store reset_counter in session
  * Updated reset_all to increment RESET_COUNTER
  * Updated API endpoint to check reset_counter
- templates/game_over.html (NEW):
  * Friendly "Game Over" page
  * Shows reason (server restart vs game reset)
  * "Join New Game" button
  * Clean, professional design

**How It Works:**
```
Server starts: RESET_COUNTER = 0
Team joins: session['reset_counter'] = 0 ✅

Host clicks Reset All: RESET_COUNTER = 1
Team tries to play: session['reset_counter'] = 0, current = 1 ❌
Result: Game Over page shown, redirect to join
```

**Testing:**
✅ RESET_COUNTER initializes to 0
✅ reset_all increments counter
✅ team_session_valid checks counter
✅ API endpoint checks counter
✅ Game Over page created
✅ No syntax errors

BUG #2: "Put Server to Sleep" doesn't stop auto-refresh
--------------------------------------------------------
**Problem:**
- Settings page had "Put Server to Sleep" button
- Only stopped timers on settings page
- Team phones kept polling API
- Didn't actually reduce server load

**Fix:**
- Added server_sleep setting to database
- Created /host/toggle-sleep API endpoint
- Created /host/get-sleep-status API endpoint
- Updated check_round_status to return sleep_mode response
- Updated team polling to stop when sleep_mode = true
- Settings page now calls API instead of localStorage

**Files Modified:**
- app.py:
  * Added 'server_sleep' to default_settings
  * Created toggle_sleep endpoint
  * Created get_sleep_status endpoint
  * Updated check_round_status to check sleep setting
- templates/settings.html:
  * Updated putServerToSleep to call API
  * Updated wakeServer to call API
  * Added sleep status check on page load
- templates/play.html:
  * Updated checkForRound to handle sleep_mode
  * Updated checkForNextRound (both instances) to handle sleep_mode

**How It Works:**
```
Host clicks "Put Server to Sleep":
1. POST /host/toggle-sleep
2. Sets server_sleep = 'true' in database
3. Button changes to "Wake Server"

Team phones polling:
1. GET /api/check-round-status
2. If server_sleep = 'true': return {sleep_mode: true}
3. Phone stops polling (returns early)
4. No more API requests = server can sleep

Host clicks "Wake Server":
1. POST /host/toggle-sleep
2. Sets server_sleep = 'false'
3. Page reloads
```

**Testing:**
✅ server_sleep setting added
✅ Toggle endpoint created
✅ Status endpoint created
✅ API checks sleep mode
✅ Team polling handles sleep mode
✅ No syntax errors
