Fix #1: Broadcast Message Persistence ✅ FIXED
----------------------------------------------
**Problem:**
- Message dismissed on waiting page, reappears on answer page
- New teams joining late get stale messages from 20 minutes ago
- No message expiry or age tracking

**Solution:**
- Added timestamp to all broadcast messages (server-side)
- Messages older than 60 seconds (1 minute) NOT shown to new joiners
- Changed from sessionStorage to localStorage for dismiss tracking
- Dismiss persists across page transitions
- Each message tracked by unique hash

**How It Works:**
```
Team joins game → Checks message age
If message < 1 minute old → Show it
If message > 1 minute old → Don't show (stale)
If already dismissed → Don't show (localStorage tracks it)
```

**Files Modified:**
- app.py (send_broadcast, api_broadcast_message)
- templates/play.html (all 4 broadcast polling sections)

**Testing:**
✅ Dismiss on waiting page → Stays dismissed on answer page
✅ New team joins 2 minutes after broadcast → Doesn't see it
✅ New team joins 30 seconds after broadcast → Sees it
✅ Same message shown again → Appears even if previously dismissed
