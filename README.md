# Family Feud - Pub Trivia Game

A Flask-based Family Feud game designed for pub trivia nights. Teams join via QR codes, submit answers, and compete for points.

## Features

- 🎯 Host dashboard with full game control
- 📱 Mobile-friendly team interface
- 🔒 PIN-protected host panel
- 📊 Real-time scoring and leaderboards
- 🖨️ Printable team code cards with QR codes
- 🔄 Auto-refresh for round changes
- 🎨 Professional UI with toast notifications

## Cloud Deployment (Render.com)

### Prerequisites
- GitHub account
- Render.com account (free tier works)
- Domain: pubfeud.gamenightguild.net

### Deployment Steps

1. **Push to GitHub**
   ```bash
   git init
   git add .
   git commit -m "Initial commit"
   git branch -M main
   git remote add origin https://github.com/YOUR_USERNAME/family-feud.git
   git push -u origin main
   ```

2. **Create Render App**
   - Go to dashboard.render.com
   - "New" → "Web Service"
   - Connect your GitHub repo
   - Render auto-detects `render.yaml`
   - Click "Create Web Service"

3. **Configure Custom Domain**
   - In Render dashboard → your app
   - "Settings" → "Custom Domain"
   - Add `pubfeud.gamenightguild.net`
   - Render provisions SSL automatically

4. **Setup DNS at GoDaddy**
   - Go to DNS Management for gamenightguild.net
   - Add CNAME record:
     - Name: `pubfeud`
     - Value: `your-app-name.onrender.com` (from Render)
     - TTL: 1 hour
   - Wait 10-60 minutes for propagation

5. **First Deploy**
   - Render auto-deploys from GitHub
   - Visit `https://pubfeud.gamenightguild.net`
   - Login with PIN (default: 6551)
   - Configure QR base URL in settings

### Environment Variables

Set in Render dashboard (already in `render.yaml`):
- `SECRET_KEY` - Auto-generated by Render
- `HOST_PIN` - Your host login PIN (default: 6551)
- `RENDER` - Set to "true" (enables cloud mode)

### Database Strategy

**Database resets on each deploy - THIS IS A FEATURE!**
- Fresh start every week
- No persistent data needed
- Each pub night = new game
- SQLite on ephemeral filesystem = perfect for this use case

## Game Flow

1. **Setup** (Host)
   - Create rounds and set answers
   - Generate team codes
   - Print code cards with QR codes

2. **Team Registration**
   - Teams scan QR or visit join page
   - Enter code and team name

3. **Game Play**
   - Activate round
   - Teams submit answers via mobile
   - Close submissions

4. **Scoring**
   - Review submissions
   - Check matching answers
   - Award points

5. **Winner**
   - View leaderboard
   - Announce winner
   - Advance to next round

## Configuration

### Settings (via /host/settings)
- QR Base URL (auto-detects cloud vs local)
- Team registration (enable/disable)
- System pause (freeze all actions)
- Broadcast messages (to all teams)

### Team Codes
- Auto-generated (30 by default)
- 4 uppercase letters (no I/O to avoid confusion)
- Printable in portrait or landscape

### Rounds
- Create manually or bulk upload
- Set question and number of answers
- Configure answer text and counts
- Activate one at a time
